# Feature 25: Portable Snapshot Report

## Implementation Summary

Extension-side fully implemented in v1.3.4. Server-side `report_handler.dart` deferred (marked optional).

**New files:** `report-types.ts`, `report-collector.ts`, `report-css.ts`, `report-scripts.ts`, `report-html.ts`, `report-commands.ts`, `report-html.test.ts`, `report-collector.test.ts` — all under `extension/src/report/` and `extension/src/test/`.

**Modified:** `extension-commands.ts` (wiring), `package.json` (command, menu, 3 config properties).

**Features:** QuickPick table selector with auto-deselect for large tables (≥10k rows), inlined CSS/JS/data, light/dark theme, client-side pagination (50 rows/page), search/filter, schema SQL blocks, anomaly severity list, XSS-safe escaping, configurable max rows/schema/anomalies via settings.

## What It Does

Export a database snapshot as a self-contained HTML file — the web UI with all data baked in, no server required. Open it in any browser, attach it to a bug report, share in Slack, or archive for later. One file, zero dependencies.

## User Experience

1. Command palette → "Saropa Drift Advisor: Export Portable Report" or tree view toolbar icon
2. A configuration dialog appears:

## Architecture

### Data Collection

The extension collects all data before assembling the HTML:

```typescript
interface IReportData {
  generatedAt: string;
  serverHost: string;
  tables: IReportTable[];
  schema?: IReportSchema[];
  anomalies?: IAnomalyReport;
}

interface IReportTable {
  name: string;
  columns: ColumnMetadata[];
  rows: Record<string, unknown>[];
  totalRowCount: number;
  truncated: boolean;       // true if totalRowCount > rows.length
}

interface IReportSchema {
  table: string;
  sql: string;              // CREATE TABLE statement
}

class ReportCollector {
  constructor(private readonly _client: DriftApiClient) {}

  async collect(config: IReportConfig): Promise<IReportData> {
    const meta = await this._client.schemaMetadata();
    const tables: IReportTable[] = [];

    for (const table of config.tables) {
      const result = await this._client.sql(
        `SELECT * FROM "${table}" LIMIT ${config.maxRows}`
      );
      const countResult = await this._client.sql(
        `SELECT COUNT(*) as cnt FROM "${table}"`
      );
      const totalCount = countResult.rows[0].cnt as number;

      tables.push({
        name: table,
        columns: meta.tables.find(t => t.name === table)?.columns ?? [],
        rows: result.rows,
        totalRowCount: totalCount,
        truncated: totalCount > config.maxRows,
      });
    }

    // Schema SQL
    let schema: IReportSchema[] | undefined;
    if (config.includeSchema) {
      const dump = await this._client.schemaDump();
      schema = this._parseSchemaStatements(dump);
    }

    // Anomalies
    let anomalies: IAnomalyReport | undefined;
    if (config.includeAnomalies) {
      anomalies = await this._client.anomalies();
    }

    return {
      generatedAt: new Date().toISOString(),
      serverHost: this._client.baseUrl,
      tables,
      schema,
      anomalies,
    };
  }
}
```

### HTML Template

The portable report is a single HTML file with all CSS, JS, and data inlined:

```typescript
class ReportHtmlBuilder {
  build(data: IReportData): string {
    return `<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <title>Saropa Drift Advisor Report — ${data.generatedAt}</title>
  <style>${this._css()}</style>
</head>
<body>
  <div id="app">
    <header>
      <h1>Saropa Drift Advisor Report</h1>
      <p>Generated: ${esc(data.generatedAt)} from ${esc(data.serverHost)}</p>
      <button onclick="toggleTheme()">Toggle Theme</button>
    </header>

    <nav id="table-list">
      ${data.tables.map(t => `
        <button class="table-btn" onclick="showTable('${esc(t.name)}')">
          ${esc(t.name)} <span class="badge">${t.totalRowCount}</span>
          ${t.truncated ? '<span class="truncated">(truncated)</span>' : ''}
        </button>
      `).join('')}
    </nav>

    <main id="table-view"></main>

    ${data.schema ? `<section id="schema">${this._renderSchema(data.schema)}</section>` : ''}
    ${data.anomalies ? `<section id="anomalies">${this._renderAnomalies(data.anomalies)}</section>` : ''}
  </div>

  <script>
    const DATA = ${JSON.stringify(data.tables)};
    ${this._js()}
  </script>
</body>
</html>`;
  }

  private _css(): string {
    // Self-contained CSS with light/dark theme support
    return `
      :root { --bg: #fff; --fg: #1a1a1a; --border: #ddd; --accent: #0066cc; }
      [data-theme="dark"] { --bg: #1e1e1e; --fg: #d4d4d4; --border: #444; --accent: #4da6ff; }
      body { font-family: system-ui, sans-serif; background: var(--bg); color: var(--fg); margin: 0; padding: 16px; }
      table { border-collapse: collapse; width: 100%; }
      th, td { border: 1px solid var(--border); padding: 6px 10px; text-align: left; }
      th { background: var(--accent); color: white; position: sticky; top: 0; }
      .badge { background: var(--accent); color: white; border-radius: 10px; padding: 2px 8px; font-size: 12px; }
      .truncated { color: orange; font-size: 12px; }
      /* ... */
    `;
  }

  private _js(): string {
    // Client-side search, pagination, theme toggle
    return `
      let currentPage = 0;
      const PAGE_SIZE = 50;

      function showTable(name) {
        const table = DATA.find(t => t.name === name);
        if (!table) return;
        currentPage = 0;
        renderTable(table);
      }

      function renderTable(table) {
        const main = document.getElementById('table-view');
        const start = currentPage * PAGE_SIZE;
        const pageRows = table.rows.slice(start, start + PAGE_SIZE);
        const cols = table.columns.map(c => c.name);

        let html = '<div class="search-bar"><input type="text" id="search" placeholder="Filter..." oninput="filterRows()"></div>';
        html += '<table><thead><tr>' + cols.map(c => '<th>' + c + '</th>').join('') + '</tr></thead>';
        html += '<tbody>' + pageRows.map(row =>
          '<tr>' + cols.map(c => '<td>' + esc(row[c]) + '</td>').join('') + '</tr>'
        ).join('') + '</tbody></table>';
        html += renderPagination(table.rows.length);
        main.innerHTML = html;
      }

      function toggleTheme() {
        document.documentElement.toggleAttribute('data-theme',
          !document.documentElement.hasAttribute('data-theme'));
      }

      function esc(v) { /* HTML escape */ }

      // Show first table by default
      if (DATA.length > 0) showTable(DATA[0].name);
    `;
  }
}
```

### Server-Side: Report Handler (Optional)

For users who want to export directly from the web UI:

```dart
class ReportHandler {
  final ServerContext _ctx;

  /// GET /api/report?tables=users,orders&maxRows=1000&schema=true&anomalies=true
  Future<void> handle(HttpRequest request, HttpResponse response) async {
    final tableNames = request.uri.queryParameters['tables']?.split(',')
        ?? await ServerContext.getTableNames(_ctx.instrumentedQuery);
    final maxRows = int.tryParse(
      request.uri.queryParameters['maxRows'] ?? '1000',
    ) ?? 1000;

    // Collect data (same logic as extension-side collector)
    final data = await _collectData(tableNames, maxRows);
    final html = _buildHtml(data);

    response.headers
      ..contentType = ContentType.html
      ..set('Content-Disposition', 'attachment; filename="drift-report.html"');
    response.write(html);
    await response.close();
  }
}
```

## Wiring in extension.ts

```typescript
context.subscriptions.push(
  vscode.commands.registerCommand('driftViewer.exportReport', async () => {
    const meta = await client.schemaMetadata();
    const allTables = meta.tables.map(t => t.name).filter(n => !n.startsWith('sqlite_'));

    // Quick pick for table selection
    const selected = await vscode.window.showQuickPick(
      allTables.map(t => {
        const info = meta.tables.find(m => m.name === t)!;
        return {
          label: t,
          description: `${info.rowCount} rows`,
          picked: info.rowCount < 10000,  // auto-deselect large tables
        };
      }),
      { canPickMany: true, placeHolder: 'Select tables to include' }
    );
    if (!selected || selected.length === 0) return;

    const config = vscode.workspace.getConfiguration('driftViewer.report');
    const collector = new ReportCollector(client);
    const data = await vscode.window.withProgress(
      { location: vscode.ProgressLocation.Notification, title: 'Collecting data…' },
      () => collector.collect({
        tables: selected.map(s => s.label),
        maxRows: config.get('defaultMaxRows', 1000),
        includeSchema: config.get('includeSchema', true),
        includeAnomalies: config.get('includeAnomalies', true),
      })
    );

    const builder = new ReportHtmlBuilder();
    const html = builder.build(data);

    const uri = await vscode.window.showSaveDialog({
      defaultUri: vscode.Uri.file(`drift-report-${new Date().toISOString().slice(0, 10)}.html`),
      filters: { 'HTML': ['html'] },
    });
    if (!uri) return;

    await vscode.workspace.fs.writeFile(uri, Buffer.from(html, 'utf-8'));
    vscode.window.showInformationMessage(`Report saved: ${uri.fsPath}`, 'Open in Browser')
      .then(action => {
        if (action) vscode.env.openExternal(uri);
      });
  })
);
```

## Known Limitations

- Large datasets produce large HTML files (10,000 rows × 10 columns ≈ 5MB HTML)
- No server-side filtering — all selected data is embedded
- Images/BLOBs are not rendered (shown as "[BLOB]")
- Client-side search is linear (no index) — slow for 10,000+ rows
- No chart/visualization support in the portable report (text tables only)
- No incremental export — entire report is regenerated each time
- The HTML template is a simplified version of the web UI — not feature-parity
- No password protection or encryption on the exported file

## Finish Report (2026-06-12)

Completes the server-side half of this feature, which the original 2026-03-14
implementation deferred as "optional" (Implementation Summary, line 5). The
extension could export a portable report since v1.3.4, but the website/server
had no equivalent — a developer using the browser viewer had no way to produce
a shareable, self-contained snapshot. A new `GET /api/report` endpoint closes
that gap and brings the website to parity with the extension command.

**What it does.** The endpoint streams a single self-contained HTML file
(`Content-Disposition: attachment`) with the database snapshot, schema DDL, and
anomaly findings inlined — no server, no network, no dependencies to open it.
The document carries a table list with row counts, click-to-view paged tables
(50 rows/page) with a per-table text filter, the schema DDL, a severity-coded
anomaly list, and a light/dark theme toggle.

**Endpoint.** `GET /api/report?tables=a,b&maxRows=N&schema=false&anomalies=false`.
`tables` defaults to every table; `maxRows` defaults to 1000 and is clamped to
1..50000 (every row is inlined, so the ceiling keeps the file shareable);
`schema`/`anomalies` set to `false` omit those sections.

**Files.**

- **`lib/src/server/report_html.dart`** (new) — `ReportHtmlBuilder.build(...)`
  plus the `ReportTableData` slice type. The report embeds the snapshot as a
  single JSON island and renders it client-side with `textContent` (never
  `innerHTML`), so a cell containing `<script>` becomes literal text — the
  report is XSS-safe by construction. The one escape applied to the JSON string
  is the standard `</` → `<\/` break-out guard so a value containing
  `</script>` cannot terminate the data tag.
- **`lib/src/server/report_handler.dart`** (new) — `ReportHandler.handle(...)`
  collects per-table columns (`PRAGMA table_info`), a capped page of rows
  (`SELECT * … LIMIT`), and the true count (`SELECT COUNT(*)`); pulls schema DDL
  via `ServerUtils.getSchemaSql`; and runs the anomaly scan via the shared
  `AnalyticsHandler` (a failing scan logs and ships an empty anomaly list rather
  than sinking the export). Requested table names are intersected with the live
  `getTableNames()` set, so the name interpolated into `SELECT` is always a real
  table — no injection surface, matching how the rest of the server treats table
  names.
- **`lib/src/server/router.dart`** — registers `_report` and routes
  `GET /api/report` inside the existing schema/export route group.
- **`lib/src/server/server_constants.dart`** — `pathApiReport`(+Alt), the four
  query-param keys, `valueFalse`, and `jsonKeyAnomalies`.
- **`lib/src/server/html_content.dart`** — a **Report** link in the web viewer's
  Export panel (and a narrative line), matching the existing hardcoded export
  links (the web-viewer HTML is not yet ARB-localized; Feature 75 web l10n is
  scaffolding only).

**Testing.**

- `test/report_html_test.dart` (6 cases) — complete-document structure, JSON
  embedding, the `</script>` break-out guard (a cell carrying
  `</script><script>alert(1)</script>` round-trips as data and never adds a
  third closing tag), schema/anomaly section toggling, and a zero-table report.
- `test/report_handler_test.dart` (4 cases, real `DriftDebugServer`) — HTML
  attachment with all tables embedded, `?tables=` restriction, unknown-table
  allow-listing, and `schema=false` omission.
- `dart test` → **621 passing**. `dart analyze` on all changed + new files →
  **No issues found** (saropa_lints clean).

**Known limitations (unchanged from the original spec).** Large datasets produce
large files (all rows inlined); client-side filter is linear; BLOBs render as
their stringified value; no charts. The `maxRows` ceiling bounds the worst case.
