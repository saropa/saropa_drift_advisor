# v4.0.4 release commit — pre-commit format hook failure

The automated v4.0.4 release commit aborted because the husky pre-commit hook ran
`dart format`, reformatted `lib/src/server/analytics_handler.dart` on disk, and exited
non-zero because an unformatted file was detected. The reformat was written to the working
tree but never re-staged, so the staged blob remained unformatted and every retry failed
identically.

## Finish Report (2026-06-18)

### Defect

- **Symptom:** `git commit failed (code 1)` during `[INFO] Committing v4.0.4...`. Hook output:
  `Formatted lib\src\server\analytics_handler.dart` / `Formatted 91 files (1 changed)` /
  `Dart formatting issues detected! Run 'dart format .' to fix.`
- **Root cause:** The pre-commit hook formats the working tree and fails when any file needed
  reformatting. It does not auto-stage its corrections. `analytics_handler.dart` was staged in
  an unformatted state as part of the release payload; the hook fixed the working-tree copy but
  the index still held the unformatted blob, so the commit blocked on every attempt.
- **Not a code defect:** the only change to `analytics_handler.dart` versus the prior staged
  state is `dart format` whitespace. No logic, control flow, or API surface changed.

### Resolution

- Staged the reformatted `lib/src/server/analytics_handler.dart` so the index matches the
  formatter's output, clearing the hook's "issues detected" condition.
- Completed the v4.0.4 release commit with the deliberately staged payload only
  (CHANGELOG, version bumps in `pubspec.yaml` / `extension/package.json` /
  `extension/package-lock.json`, `analysis_options.yaml`, `server_constants.dart`,
  `add-package.ts`, and the new `bugs/BUG_lint_report.md`). Unrelated working-tree edits in
  other server source files were left unstaged as a separate workstream.

### Prevention note

A pre-commit formatter that mutates files but exits non-zero will block any release flow that
stages a snapshot before the hook runs. Either run `dart format .` before staging in the
release script, or have the hook re-add files it reformats. Not changed here to keep the fix
scoped to the reported symptom.
