# Open Database Browser ΓÇõ discoverability

The command that opens the live database web viewer (`driftViewer.openInBrowser`,
which launches `http://host:port` in an external browser) existed but was reachable
only from the Database Explorer view's "ΓÇª" overflow menu and from disconnected-state
welcome screens ΓÇõ users reported being unable to find how to open the browser from
the extension or its dashboards. The fix surfaces the existing command in two
always-visible places without adding new commands or message routes.

## Finish Report (2026-06-24)

### Scope
VS Code extension (TypeScript + `package.json` manifest) and CHANGELOG. No Dart/Flutter
app code. No new commands, dependencies, or webview scripts.

### Defect
`driftViewer.openInBrowser` was contributed to the `view/title` menu under the
`1_explore@1` group, which VS Code renders inside the overflow ("ΓÇª") menu rather than
as an inline toolbar icon. The Drift Tools hub hero exposed a "Rescan" and an
"Open website" (saropa.com) action but no action to open the database web viewer. The
result was that the single most-wanted action ΓÇõ open the live viewer ΓÇõ had no visible
affordance on either the sidebar or the hub dashboard.

### Change
1. `extension/package.json` ΓÇõ moved the `openInBrowser` contribution from
   `view/title` group `1_explore@1` to the inline `navigation` group at order `@3`
   (between the table-grouping toggle and the Dashboard icon), and renumbered the
   trailing inline icons (`openDashboard` 3ΓÁÈ4, `healthScore` 4ΓÁÈ5, `askNaturalLanguage`
   5ΓÁÈ6, `showToolsQuickPick` 6ΓÁÈ7). The globe icon now shows inline whenever
   `driftViewer.serverConnected`.
2. `extension/src/hub/hub-html.ts` ΓÇõ added a primary hero button labeled via
   `panel.hub.btn.browser`, wired through the existing validated `runCommand`
   postMessage path with `data-cmd-id="driftViewer.openInBrowser"`. No new message
   handler was needed: `DriftToolsHubPanel._handleMessage`'s `runCommand` case already
   validates the `driftViewer.` prefix before `executeCommand`. The pre-existing
   "Open website" button is retained (demoted from primary to default styling so the
   browser action reads as the lead CTA). The `database` hub icon is used (the hub icon
   set has no globe).
3. `extension/src/l10n/strings-panel-hub.ts` ΓÇõ added the host English source string
   `panel.hub.btn.browser` = "Open Database Browser". `t()` resolves symbolic key ΓÁÈ
   English source ΓÁÈ `vscode.l10n.t`; the extension ships English-only, so no bundle
   regeneration is required. Translation remains a separate pipeline.
4. `extension/src/test/hub-html.test.ts` ΓÇõ added an assertion pinning the primary
   hero button and its `openInBrowser` wiring so the affordance cannot silently
   regress.
5. `extension/CHANGELOG.md` ΓÇõ added an `[Unreleased]` section (Added: hub button;
   Changed: toolbar icon promotion).
