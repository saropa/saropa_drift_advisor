# Feature 73: Website Bulk Index Creation

**Status: PLANNED** — split out of the archived GAP parity analysis (originally
WP-A05 / section 11 row "Bulk index creation"). The website applies index
suggestions one at a time; this adds multi-select batch apply with SQL preview.

Source: [GAP_FIT_PLAN.md (archive)](./history/2026.06/2026.06.10/GAP_FIT_PLAN.md) §11.

## Gap

| Surface | Capability            | State       |
| ------- | --------------------- | ----------- |
| W       | Bulk index creation   | **missing** |

## Tasks

- Add a multi-select control to the index suggestion list.
- Generate a merged SQL preview with per-index status.
- Apply as one action with partial-failure reporting.
- Provide rollback guidance (or generated rollback SQL when available).

## Exit criteria

- User can select 2+ suggestions, preview the merged SQL, apply in one action,
  and validate the resulting schema/indexes; partial failures are reported.

## Implementation Plan

### Current reality (correction to the original gap framing)

The website does **not** apply index suggestions "one at a time" — it does not
apply them at all. Suggestions are display-only: the SQL is rendered in a table
and clicking a cell copies it to the clipboard for manual paste
(`assets/web/tools-analytics.ts` → `initIndexSuggestions()` / `renderIndexData()`;
`#index-analyze` button in `lib/src/server/html_content.dart`). So this feature
adds an apply path *and* makes it batch from the start.

- Suggestions are generated by `IndexAnalyzer.getIndexSuggestionsList()`
  (`lib/src/server/index_analyzer.dart`); each is
  `{table, column, reason, sql, priority}`.
- Served at `GET /api/index-suggestions` via
  `AnalyticsHandler.handleIndexSuggestions()`
  (`lib/src/server/analytics_handler.dart`), registered in `_routeAnalyticsApi()`
  (`lib/src/server/router.dart`).
- A batch executor already exists for data mutations:
  `EditsBatchHandler.runValidatedBatchStatements()`
  (`lib/src/server/edits_batch_handler.dart`) wraps statements in
  `BEGIN IMMEDIATE; … COMMIT;` and reports per-statement failure via
  `BatchApplyStatementError` (index + statement + cause). **But** the validator
  (`SqlValidator.isSingleDataMutationSql()` in
  `lib/src/server/sql_validator.dart`) currently *rejects* `CREATE INDEX`.

### Phase 1 — Validator: allow CREATE INDEX

- Add `SqlValidator.isSingleCreateIndexSql()` matching
  `^CREATE\s+(UNIQUE\s+)?INDEX\s+(IF\s+NOT\s+EXISTS\s+)?…`, single-statement,
  no stacked DDL/PRAGMA. Keep it separate from the data-mutation validator so
  index DDL is gated by an explicit, narrowly-scoped check.

### Phase 2 — Merged SQL preview endpoint

- `POST /api/indexes/preview` (new constant in `server_constants.dart`): input
  `{indexSqls: [...]}`, output the validated, merged statement list (and reject
  reasons for any that fail validation). No DB write — preview only.

### Phase 3 — Batch apply with per-index status

- Reuse `runValidatedBatchStatements()` but swap in the new CREATE-INDEX
  validator; surface a per-index result array
  `[{index, sql, ok, error?}]` rather than failing the whole batch opaquely.
  Decide transaction semantics explicitly: all-or-nothing (current
  `BEGIN…COMMIT`) vs. best-effort per-index — recommend all-or-nothing first
  (simplest correct behavior), with partial-apply as a follow-up if requested.
- Route under `_routeWriteApi()` (write-enabled only, matching the existing
  `POST /api/edits/apply` gating).

### Phase 4 — Multi-select UI

- Extend `renderIndexData()` in `assets/web/tools-analytics.ts`: add a checkbox
  per suggestion, a selection count, a "Preview SQL" panel (calls Phase 2), and
  an "Apply selected" button (calls Phase 3, write-enabled only).
- Render the per-index result (green/red + error text) after apply; offer retry
  of only the failed rows.

### Exit gate

- Select 2+ suggestions, preview the merged SQL, apply in one action, see
  per-index success/failure, and confirm the created indexes exist in the live
  schema. Verify a deliberately bad suggestion reports its own failure without
  silently dropping the others.
